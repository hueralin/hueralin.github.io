敏捷宣言

12条原则

Scrum

XP(极限编程)
1. 结对编程
2. 测试驱动开发：单元测试，重构
3. 持续集成
4. 代码集体所有（统一规范，任何人写的代码，任何人都能维护）
5. 现场客户

精益思维
1. 消除浪费
2. 增强学习
3. 尽量延迟决定
4. 尽快发布
5. 下放权力
6. 嵌入质量
7. 全局优化

MoSCoW

估算
